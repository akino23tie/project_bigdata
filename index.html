Latar Belakang & Masalah

Mengapa sistem monitoring otomatis diperlukan untuk lahan gambut?

Urgensi Bencana

Lahan gambut rentan terhadap dua kondisi ekstrem: Kebakaran saat kering (suhu tinggi, kelembaban rendah) dan Banjir saat musim hujan (muka air naik).

Tantangan 5V Big Data

Sensor mengirim data secara kontinu (Velocity) dengan jumlah besar (Volume), yang sulit diolah secara manual untuk mendapatkan keputusan cepat (Value).

Otomasi EWS

Diperlukan sistem yang mampu mengolah data sensor (Muka air, Suhu, Kelembaban, Cahaya) menjadi peringatan dini status "Normal", "Waspada Kebakaran", atau "Waspada Banjir".

Arsitektur Pipeline Big Data

Pipeline Big Data merupakan serangkaian tahapan untuk mengalirkan data dari sumber hingga menjadi informasi. Alur ini mencakup Ingestion, Processing, Storage, dan Visualization untuk memantau kondisi lahan gambut.

1. Data Ingestion

Google Colab (Python)

Script simulasi mengumpulkan data Sensor Suhu & Cahaya (simulasi IoT) dalam format CSV.

2. Data Processing

Python (Pandas)

Proses cleaning data, penanganan missing value, dan standarisasi format sebelum disimpan.

3. Data Storage

MongoDB

Menyimpan data hasil olahan ke dalam database NoSQL (Document Store) agar siap diakses secara fleksibel.

4. Data Visualization

Tableau

Dashboard interaktif menampilkan grafik tren suhu dan intensitas cahaya sebagai sistem peringatan dini.
